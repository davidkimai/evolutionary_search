# Domains Summary

The harness is organized into four domains:

- **architecture** — the codex-native runtime, TinyFish integration, schemas, and repo shape
- **product** — the project frame, wedge, user journeys, and scoring model
- **execution** — the phase plan, packets, state tracking, and definition of done
- **demo** — judge mapping, claims ledger, and presentation script

## One-sentence strategy

Build a **Codex-native evolutionary search system over the open web** that uses TinyFish to search messy real sites in parallel, then ranks evidence-backed opportunities.
