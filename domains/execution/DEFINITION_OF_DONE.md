# Definition of Done

The project is done only when all of the following are true:

- app-server-native search sessions run locally
- `thread/start`, `turn/start`, and streamed events are actually used in the product flow
- TinyFish live mode works or is honestly marked blocked with replay fixtures fully replacing it for demo
- replay mode is deterministic and judge-safe
- the opportunity wedge returns ranked, evidence-backed results
- scoring is decomposable and visible
- web demo shell works
- CLI recovery surface works
- tests/evals pass
- `review/start` or equivalent review flow has been used to harden the final state
- docs and claims ledger reflect reality
