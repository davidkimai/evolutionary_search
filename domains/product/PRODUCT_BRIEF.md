# Product Brief

## Name

Primary project name:
**Open Web Evolutionary Search**

Recommended short references:
- **Evolutionary Search**
- **OWES** only if an acronym is truly useful internally

Do **not** lead externally with “Opportunity Engine” anymore.
That is now a wedge application, not the system name.

## Why this framing is better

### Better for hackathon judging
- more technically distinctive
- matches the HH paradigm directly
- creates a stronger conceptual “aha”
- fits a Codex-native project better than a conventional product noun

### Still good for PMF
The risk is that “Evolutionary Search” is more abstract.
Solve that by pairing it with the concrete first wedge:

> Open Web Evolutionary Search for startup program search across grants, cloud credits, and accelerators.

That gives both technical novelty and business clarity.

## First wedge

Build the MVP around startup program search:
- grants
- cloud credit programs
- accelerators

## Product promise

Given a profile and an objective, the system searches fragmented live websites in parallel, verifies evidence, and returns a ranked shortlist with next actions. That loop can later generalize to other messy, high-value selection problems across the open web.
