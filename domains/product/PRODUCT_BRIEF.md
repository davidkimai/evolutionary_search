# Product Brief

## Name

Primary project name:
**Open Web Evolutionary Search**

Recommended short references:
- **Evolutionary Search**
- **OWES** only if an acronym is truly useful internally

Do **not** lead externally with “Opportunity Engine” anymore.
That is now the wedge, not the system name.

## Why this framing is better

### Better for hackathon judging
- more technically distinctive
- matches the HH paradigm directly
- creates a stronger conceptual “aha”
- fits a Codex-native project better than a conventional product noun

### Still good for PMF
The risk is that “Evolutionary Search” is more abstract.
Solve that by pairing it with the concrete wedge:

> Open Web Evolutionary Search for grants, accelerators, and tenders.

That gives both technical novelty and business clarity.

## Core wedge

Build the MVP around:
- grants
- accelerators
- public tenders / RFPs

## Product promise

Given a profile and an objective, the system searches fragmented live websites in parallel, verifies evidence, and returns a ranked shortlist with next actions.
